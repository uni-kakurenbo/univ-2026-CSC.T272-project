### ビジュアライザ・クライアント（React）の実装計画

情報非対称性とAIの推論過程（ベイズ分布）を「直感的に理解・説明できるプレゼンツール」として作り込みます。

#### 🛠️ コア実装スペック (React + Vite)

- **推奨ライブラリ:** `Tailwind CSS` (スタイリング), `Lucide React` (アイコン), `HTML5 Canvas` または `CSS Grid` (グリッド描画), `Recharts` (収束グラフ描画)
- **主な責務:**

1. **多視点グリッドレンダリング:**

- **「神の視点 (God View)」:** 全壁、真の座標、全ステータスを表示。
- **「二号の視点 (Duck View)」:** 視界外を半透明の黒（フォグ）で覆い、二号が telemetry として返したスフィンクス位置の確率分布を**マスの赤色ヒートマップ**として可視化する。
- **「スフィンクスの視点 (Sphinx View)」:** 地形全体と真のゴールを表示しつつ、スフィンクスが telemetry として返した二号位置の確率分布を**マスの青色ヒートマップ**、ゴール位置の確率分布を**マスの黄色ヒートマップ**として重ね合わせる。真のゴールはエージェントへの観測には含めず、推定結果との比較用としてビジュアライザだけに表示する。
- ヒートマップのマスをホバーすると、その座標と確率をツールチップに表示する。相手位置とゴールの確率分布が重なるマスでは両方の確率を表示する。
- 両視点で最尤予測セルを、ヒートマップ上でも明瞭に見える白い `P` の破線枠として表示する。移動は盤面上に重ねた軌跡レイヤーで、直近数手の実際の始点・終点を太い半透明の直線として表示し、古い軌跡ほど薄くする。真のゴールは `G` のトークンとして表示するため、真値と推定値が同じマスでも識別できる。サイドパネルには各移動ステップの「要求方向 → 実移動方向」、スリップ判定後の解決方向、始点・終点、結果を表示する。

2. **タイムライン・プレイヤー:** 「再生 / 一時停止 / コマ送り / コマ戻し / 再生速度調整（0.5x 〜 10x）」ができるシークバー機能。
    - **ライブ観戦:** クライアントが `/ws/view` で現在の試合のフレームをメモリ上にバッファする。新しい試合が開始される（`{type:"status", state:"waiting-for-agents"}` を受信する）たびにライブ用バッファをリセットする。複数試合が並列実行されている場合は、フレーム混在を避けるため最初に受信した `matchId` のフレームだけをライブ盤面に反映する。
    - **過去対戦:** `GET /api/matches` で保存済み対戦の一覧を表示し、選択時に `GET /api/matches/:id` から全フレームを取得する。同じタイムライン・プレイヤーへ渡すことで、ライブと過去対戦で再生処理を共通化する。過去対戦の表示中もライブ用バッファの受信は継続し、「ライブに戻る」で最新状態へ復帰できる。
3. **リアルタイム分析パネル:** 各エージェントの相手位置予測・確信度を表示する。値はサーバーが真の位置から合成せず、エージェントが行動応答で返した telemetry をそのまま使用する。
4. **対戦実行コントロール:** Server（`docs/visualizer/server.md` 参照）へ `POST /api/matches` で新規試合の開始を要求する（`apps/visualizer/src/components/MatchControls.tsx`）。実行数入力で `count` を指定でき、対戦中でも追加投入できる。Server 側の同時実行上限を超えた分はキューで待機し、ボタンには実行中・待機中・同時実行上限を表示する。
5. **対戦履歴:** 完了日時・勝者・ターン数を新しい順に表示する。保存済み履歴全体から総試合数、二号勝率、スフィンクス勝率、引き分け率を表示し、不要になった履歴は全削除できる。選択した履歴はリプレイであることを明示し、ライブ観戦へ戻る導線を常に表示する。

#### 📋 実装ロードマップ

- **Step 1:** WebSocketで受信する状態JSONの型定義（TypeScript interface）を合意・作成する。
- **Step 2:** 単純な二次元配列を受け取って、色分けされたグリッドとキャラクターを描画する `DungeonBoard` コンポーネントを作成する。
- **Step 3:** 視点切り替えタブ（God / Duck / Sphinx）と、確率値を背景色の濃淡（`rgba(255, 0, 0, opacity)` 等）で表現するヒートマップオーバーレイ関数を実装する。
- **Step 4:** 再生コントローラーと、グラフライブラリを用いた推移チャートを統合する。
