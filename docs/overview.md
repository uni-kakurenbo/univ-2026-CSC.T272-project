# プロジェクト全体概要書

**不確実・視野制限環境における非対称情報POMDP対戦プラットフォームの構築と評価**
**〜 ホイールダック2号（ステルス脱出） vs スフィンクス（敵対的確率探索） 〜**

## 1. プロジェクトの目的と背景

本プロジェクトは、実世界の自律移動ロボットが直面する「観測の不確実性」「行動の制御誤差」「敵対的エージェントの存在」という課題に対し、人工知能講義で習得した数理モデルを統合・実装して解決を試みる共同研究です。

同じダンジョン問題に対し、２人の開発者が「視野は狭いがセンサー精度の高い脱出エージェント（二号）」**と**「ゴールを除く地形全体を知り、二号の位置を距離依存の誤差付きで観測する追跡エージェント（スフィンクス）」**の視点に分かれて独立にアルゴリズムを構築します。
これにより、単なる経路探索にとどまらない**非対称情報ゲーム（Asymmetric Information Game）および部分観測マルコフ決定過程（POMDP）における高度な心理・戦略戦をシミュレーションし、各理論モデルの有効性を定量的に検証することを目的としています。

## 3. システムアーキテクチャ全体像

お互いのコードに干渉せず並行開発を進め、かつ高速なモンテカルロシミュレーションとリッチな画面描画を両立させるため、「競技AIコンテスト方式（プロセス分離型）」の分散システムを採用しています。

```
+-----------------------------------------------------------------------------------+
| [プレゼンテーション層] React + Tailwind CSS + Vite                                 |
|  - 3視点同時描画 (神の視点 / 二号の霧と危険度マップ / 敵の推論可視化)       |
+-----------------------------------------+-----------------------------------------+
                                          ^
                                          | (WebSocket: リアルタイム状態 JSON)
+-----------------------------------------v-----------------------------------------+
| [Server] Bun (Bun.serve / TypeScript)                                           |
|  - API / 観戦WebSocket / Runnerプロセス管理 / 履歴保存・勝率集計                       |
+-----------------------------------------+-----------------------------------------+
                                          |
                                          | (Runner stdout + Adaptor WebSocket)
         +--------------------------------+--------------------------------+
         v (標準入力: 観測 JSON)                                           v (標準入力: 観測 JSON)
+---------------------------------+                              +---------------------------------+
| 二号 AI エージェント (C++ バイナリ) |                              | スフィンクス AI (C++ バイナリ)   |
|  - [脱出担当者が開発]             |                              |  - [追跡担当者が開発]             |
+---------------------------------+                              +---------------------------------+
         | (標準出力: アクション)                                          | (標準出力: アクション)
         +-------------------------------->--------------------------------+

```

### アーキテクチャの強み

- **完全疎結合な並行開発:** C++開発者はネットワーク通信を一切気にせず、「標準入力からデータを受け取り、計算して標準出力へ行動を返す」だけの純粋なアルゴリズム実装に集中できます。
- **強力なデバッグ環境:** エージェントのデバッグ出力（`stderr`）は通信を汚すことなくコンソールに反映され、画面側（React）では内部推論確率の可視化が可能です。
- **実験の自動化:** 描画をオフにしたバッチモードでは、C++同士の総当たり対戦を爆速で回し、評価グラフ用のデータセットを瞬時に生成できます。

### Docker化と対話型ワンクリック起動

上記の「プロセス分離型」構成はDockerでは `server` / `visualizer` の2コンテナに対応付ける。`server` コンテナ内では、APIを担当する Server が試合ごとに Runner（1試合実行プロセス）と Adaptor（WebSocket ↔ 標準入出力変換）を spawn し、C++エージェントを起動する。`visualizer` はビルド済みビジュアライザを配信する。さらにビジュアライザには「試合の実行」ボタンを追加し、`POST /api/matches` 経由でクリックだけで新しい対戦を開始できる（`docs/visualizer/server.md`・`docs/deployment.md` 参照）。既存のバッチ実験・AI理論部分の設計はこれらの追加によって変更されない。

コードは `apps/server`（API / Runner 管理）・`apps/runner`（1試合実行プロセス）・`apps/adaptor`（WebSocket ↔ 標準入出力変換）・`apps/visualizer`（React + Vite）・`apps/agents`（C++エージェント）・`apps/shared`（共通型定義）からなるBunワークスペースとして構成する。

1. **生存・捕獲率マトリクス:** 上表の各セルにおける二号の生存率／スフィンクスの捕獲率を比較し、単純な経路探索（A*のみ）に対する、ベイズ推論＆ゲーム木探索を統合した提案手法の優位性を実証する。
2. **ステップ効率化分析:** 脱出・捕獲までの消費ステップ数の分布比較。

- スフィンクス側: ゴール候補の推論が真のゴールへ収束していく過程の可視化。
